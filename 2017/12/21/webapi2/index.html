.net core webapi使用swagger生成api帮助文档

此篇文章接上一篇

  • 安装依赖 Install-Package NSwag.AspNetCore

  • 在该Startup.Configure方法中，启用用于提供生成的Swagger规范和Swagger UI的中间件：

    app.UseStaticFiles();

    app.UseSwaggerUi(typeof(Startup).GetTypeInfo().Assembly, settings =>

    {
    //可在 GeneratorSettings设置相应的描述信息
    settings.GeneratorSettings.DefaultPropertyNameHandling = PropertyNameHandling.CamelCase;
    settings.GeneratorSettings.Description = "测试nswag";
    settings.GeneratorSettings.Title = "my Nswag";
    settings.GeneratorSettings.Version = "2.0";
    //settings.MiddlewareBasePath = "localhost:5000:api/*";
});
  • 右键 项目->属性 勾选xml选项

  • 修改launchSettings.json,将默认地址修改为swagger

  • 一般nswag是根据你的方法注释生成xml文档的 ,如果api方法中需要返回400 200等操作码时,应当这么写

其中 remarks节点指定一个返回体

运行生成此页面表示构建成功
